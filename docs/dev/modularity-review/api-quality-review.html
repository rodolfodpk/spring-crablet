API Quality Review — Current Status

Scope: public EventStore API, command handler API, CommandDecision, CommandExecutor, and related value types. Status date: May 2026.

Executive Summary

The original review identified several pre-1.0 polish issues around abstraction boundaries, typed command decisions, and direct event-store ergonomics. The critical boundary issues have since been fixed. The remaining items are accepted tradeoffs: executeInTransaction(...) remains public for advanced direct EventStore usage, and appendIdempotent(...) keeps both a simple raw-string overload and a Query overload.

The current API shape is coherent enough to treat as the public baseline, with focused guardrail tests to prevent the old issues from reappearing.

Current API Boundary

EventStore is the foundational application-facing abstraction for event appends, projections, existence checks, and direct transaction composition. It should not expose command-layer concerns. The public guardrail is simple: methods such as storeCommand(...) and hasConflict(...) must not appear on EventStore.

CommandExecutor owns command execution. It runs handler decision logic, dispatches the resulting CommandDecision, handles idempotent results, records command audit when enabled, and reports metrics. Its public execute(...) overloads return ExecutionResult.

Accepted Tradeoffs

Keeping EventStore.executeInTransaction(...) public broadens EventStore beyond simple event storage, but the tradeoff is useful. It gives direct EventStore users the same primitive the command layer relies on: one JDBC connection, configured isolation, atomic commit/rollback, and a transaction-scoped store for projections and appends.

The raw-string appendIdempotent(events, eventType, tagKey, tagValue) overload is intentionally kept as the common-case API. The appendIdempotent(events, idempotencyQuery) overload supports advanced idempotency criteria without forcing simple callers to build a Query.

Guardrail Tests

The API baseline should be protected by focused tests rather than a brittle snapshot of every public method: EventStore must not expose command internals, CommandExecutor.execute(...) must return ExecutionResult, commutative handlers must return CommutativeDecision, and ProjectionResult must remain passive.
